Everything outside the kernel is a module. A module is a self-contained piece that plugs in through a standard interface to add a specific capability. Six types, one universal contract, fixed loading order. Remove one, add another. The rest of the system doesn't notice.

Let's walk through the six types. The Orchestrator drives the conversation — it decides context, routing, tool handling, and when to stop. There's exactly one, and it's required. Providers connect to AI models — Claude, GPT, Gemini, a local model — translating between Amplifier's format and the model's API. Multiple are allowed. Tools are actions the agent can take: read a file, search the web, run a command. The AI decides when to use them. The Context Manager keeps track of what's been said and manages the token budget. There's exactly one, and it's required. Hook Handlers react to events, fire automatically, and are invisible to the AI. They can observe, block, inject, or modify. And finally, Resolvers map module names to file paths — infrastructure plumbing that users rarely encounter directly.

Tools are visible to the AI — they're listed in the system prompt. Hook Handlers operate invisibly in the system layer. Chapter Seven covers the full distinction between tools and hooks.

All six module types plug in through the same function: an async mount function that receives the Coordinator and a configuration dictionary. The module does its setup, registers itself in the Coordinator at the right slot, and optionally returns a cleanup function. Amplifier uses structural subtyping — if your class has the right methods with the right signatures, it qualifies. No inheritance required.

Modules load in a fixed order: Orchestrator first, then Context Manager, then Providers, then Tools, then Hook Handlers, then Resolvers on demand. Orchestrator and Context Manager are required — if either fails to mount, the session stops immediately. The rest are optional: the system warns and continues. The loading order is fixed, and the sequence matters. Orchestrator first because it's the conductor. Context Manager second because the Orchestrator needs memory. Hooks load last because hooks observe everything else. They need the full system set up before they start watching.
